\begin{abstract}
Primitive recursion is usually introduced as a class of total functions
on natural numbers, but the same idea extends to richer inductive data
and higher-order programs.  We present an Agda formalization of
primitive recursion in categorical terms, using point-free first-order
and higher-order calculi, together with equational theories, setoid
models, interpretation soundness, verified translations between
paramorphism-primitive and fold-primitive presentations, and
representative instances for natural numbers, words, trees, and
many-sorted signatures.

The formalization is organized around small core calculi.  Each hom-set
is equipped with a setoid equality containing the categorical laws,
product and coproduct laws, functoriality and strength laws, and
recursion computation and uniqueness principles.  Higher-order calculi
add exponentials and their beta/eta laws.  We use setoids rather than
quotients and keep categorical distributivity as standard infrastructure:
primitive in the first-order presentation and derived from exponentials
in the higher-order presentation.

All definitions and proofs are checked by Agda using the safe fragment.
\end{abstract}

\section{Motivation and Running Examples}
\label{sec:motivation}

Before turning to the Agda artifact, we spell out the mathematical
pattern that the artifact abstracts.  The starting point is the familiar
class of primitive recursive functions on natural numbers.  In the usual
presentation, an \(n\)-ary program denotes a function
\(\mathbb{N}^n \to \mathbb{N}\).  The basic programs are zero,
successor, and projections; larger programs are built by composition and
by a primitive-recursion constructor.

The source language \AgdaFile{PR-Nat} formalizes this grammar as an
Agda datatype \(\AgdaDatatype{PR}\ n\), where \(n\) is the arity of the
program.  Its constructor \(P(g,h)\) builds a program of arity \(1+n\).
The first input is the induction argument; \(g\) handles the zero case
using the remaining \(n\) parameters, and \(h\) handles the successor
case using the value obtained for the predecessor, the predecessor
itself, and the same parameters.  Addition is the smallest example: it
is specified by induction on the first argument,
\[
  \begin{array}{rcl}
  \mathsf{add}(0,y) &=& y,\\
  \mathsf{add}(x+1,y) &=& \mathsf{suc}(\mathsf{add}(x,y)).
  \end{array}
\]
In the syntax of \AgdaFile{PR-Nat}, the corresponding program is
generated from one projection for the base case and successor applied to
the predecessor value in the step case:
\[
  \mathsf{add}
  =
  P(\pi_0,\; \mathsf{suc} \circ \pi_0)
  \quad:\quad \mathbb{N}^2 \to \mathbb{N}.
\]
Here the first \(\pi_0\) selects the remaining argument \(y\) in the
zero case, while the second \(\pi_0\) selects the value already obtained
for \(x\) in the successor case.  The notation is already point-free:
variables are represented only through projections, tupling, and
composition.  This is the first categorical reading of ordinary
primitive recursion.  Types are objects, programs are arrows, products
account for multiple arguments, and the primitive-recursion constructor
is an arrow former with a computation law.

Once stated this way, very little is special about natural numbers.
The natural numbers are the initial algebra of the polynomial functor
\[
  F(X) = 1 + X,
\]
where the left summand represents zero and the right summand represents
successor.  The same pattern applies to any strictly positive polynomial
functor.  A list of letters from a finite alphabet \(A\), for instance,
is the initial algebra of
\[
  F(X) = 1 + A \times X.
\]
The word operation \(\mathsf{append}\) follows the same induction
pattern on the first word: the empty-word case returns the second
argument, and the letter case prefixes the current letter onto the value
obtained for the tail,
\[
  \begin{array}{rcl}
  \mathsf{append}(\epsilon,w) &=& w,\\
  \mathsf{append}(a :: u,w) &=& a :: \mathsf{append}(u,w).
  \end{array}
\]
The point-free program has the same shape as addition, except that
successor has been replaced by the family of letter constructors.  The
word source language therefore has one step handler for each leading
letter.

Ranked trees give the next example.  A signature with constructor set
\(\Sigma\) and arity function
\(\mathsf{ar} : \Sigma \to \mathbb{N}\) determines the polynomial
\[
  F(X) = \sum_{c \in \Sigma} X^{\mathsf{ar}(c)}.
\]
A primitive recursive tree program supplies one handler per constructor.
For a binary node \(c\), the handler receives one induction hypothesis
for each child, and in the paramorphism presentation it also receives
the original children.  This extra access is useful for programs that
need to rebuild or inspect the input tree while accumulating information.
The many-sorted version replaces \(X\) by a family of carrier sets
indexed by sorts and replaces a single arity with a vector of input sorts
for each constructor.

The first-order core, \PRFO{}, abstracts over these examples.  Its types
include finite products and coproducts together with inductive types
\(\mu F\) for polynomial functors \(F\).  Its arrows are categorical
combinators plus functorial action, strength, and either a primitive
paramorphism \(P\) or a primitive fold \(F\).  An instance module is a
checked translation from one concrete source language, such as
\AgdaFile{PR-Nat} or the word and tree languages, into this core.  The
translation shows that the concrete language is not a separate calculus:
it is an instance of the same point-free structure.

The next step is higher order.  Harper's textbook presentation of
System~T~\cite{DBLP:books/cu/Ha2016} extends simply typed lambda
calculus with natural numbers and a natural-number recursor.  Our
higher-order core, \PRHO{}, adds exponentials to the first-order
categorical language, yielding a point-free counterpart of function
types, abstraction, and application.  This makes it possible to connect
System~T directly to \PRHO{} and then to view simply typed lambda
calculus with polynomial inductive types as the contextual presentation
of the same structure.

\section{Core Calculi}
\label{sec:core-calculi}

The artifact is organized around four closely related point-free core
calculi.  The first-order calculi, \PRFO{} and \PRFOF{}, have finite
products, finite coproducts, polynomial inductive types, functorial
action, strength, and an eliminator for inductive types.  The
higher-order calculi, \PRHO{} and \PRHOF{}, extend the same syntax with
exponentials.  The suffix-free calculi take paramorphism as primitive;
the \(F\)-suffixed calculi take fold as primitive.

All four calculi share the same type language, parameterized by a mode.
The mode index \AgdaBound{md} prevents exponentials from appearing in first-order
types, while allowing the rest of the grammar to be reused unchanged.
The \(0\) in the first argument of the exponential constructor is
deliberate: the domain of a function type must be closed.  The index
\(n\) tracks the variables over which a type code acts functorially.
Those variables may occur in the codomain of an exponential, but
occurrences in the domain would be contravariant and would invalidate
the generic \AgdaInductiveConstructor{fmap} and
\AgdaInductiveConstructor{strength} structure.  Thus a higher-order
type operator may contain function spaces \(A \Rightarrow G\) only when
\(A\) has no free type variables.
The constructor \AgdaInductiveConstructor{ind} binds one type variable
in the code for a polynomial functor and forms its least fixed point.

{\small
\CoreTypesGrammar*
}

\subsection{First-Order Core}
\label{sec:core-calculi-fo}

The arrow syntax is intrinsic: the source and target types are indices
of the Agda datatype.  The first-order core with primitive paramorphisms
is shown below.  The constructors up to \AgdaInductiveConstructor{`case}
are the usual categorical structure for finite products and coproducts.
The constructor \AgdaInductiveConstructor{dist-+-×} is included as
first-order infrastructure.  The constructors \AgdaInductiveConstructor{fmap}
and \AgdaInductiveConstructor{strength} make polynomial functors
available at the term level, and \AgdaInductiveConstructor{P} is the
primitive paramorphism operator.  We write \AgdaBound{G}\AgdaSpace{}[
\AgdaBound{T}\AgdaSpace{}] for substituting \AgdaBound{T} for the
bound type variable in \AgdaBound{G}.

{\small
\CorePRFOSyntax*
}

The type of \AgdaInductiveConstructor{P} is the categorical version of
primitive induction.  Given an input of type \AgdaDatatype{ind}
\AgdaBound{G} and parameters \AgdaBound{U}, the step arrow receives the
one-layer unrolling of the inductive value where recursive positions
carry both the already computed result of type \AgdaBound{T} and the
original substructure of type \AgdaDatatype{ind} \AgdaBound{G}.  This is
the paramorphism distinction: a step can inspect the original recursive
subdata as well as the induction hypotheses.

The fold-primitive first-order calculus has the same categorical
constructors, but replaces \AgdaInductiveConstructor{P} by
\AgdaInductiveConstructor{F}.  Its step receives only the functorial
layer of already computed results, not the original substructures.

{\small
\CorePRFOFoldPrimitive*
}

\label{sec:core-calculi-ho}

paramorphism-primitive presentation, the two additional arrow

\section{Equational Theories and Setoid Models}
\label{sec:equations-models}

The datatype of arrows in Section~\ref{sec:core-calculi} is deliberately
intensional: two arrows are the same Agda value only when they have the
same syntax.  This is too strict for the mathematical theories.  For
example, composition with the identity should not change an arrow, and
the eliminator for products should be determined by its two projections.
Instead of quotienting the syntax, the formalization keeps raw arrows and
defines, for each calculus, an indexed equality relation generated by the
expected equations.  The first-order paramorphism calculus is
representative:

{\small
\CorePRFOEquivalence*
}

This relation is an object-language equality between parallel arrows.
Its first constructors make it an equivalence relation, and the next
constructors say that equality is preserved by the term formers.  The
remaining constructors, not all shown here, are the actual laws of the
calculus.  They include the category laws for identity and composition;
the beta and eta laws for products and coproducts; the beta and eta laws
for exponentials in the higher-order calculi; the functoriality and
strength laws for type operators; the distributivity laws in the
first-order calculus where distributivity is primitive; and the
computation and uniqueness principles for primitive recursion.  For
paramorphisms, the two characteristic laws are:

{\small
\CorePRFOParaLaws*
}

The beta law says how a paramorphism behaves after one constructor
step: the recursive call is exposed through \AgdaFunction{paraArgs},
which supplies both the recursive results and the original subdata to
the step function.  The uniqueness law gives the corresponding induction
principle for arrows out of an inductive type.  Thus recursion is not
specified by an evaluator external to the syntax; it is specified by the
equations that characterize the primitive arrow.

The semantic side follows the same separation between raw structure and
equality.  A structure provides an interpretation of the object-language
types as objects and of each primitive arrow former as an operation on
semantic arrows.  A model adds a relation
\(\AgdaOperator{\AgdaField{\_≈ᴹ\_}}\) on every hom-type, proves that this
relation is an equivalence, and validates each generated law.  The
interpretation function is defined by recursion on raw syntax, while the
soundness theorem maps a derivation \(f \approx g\) in the generated
equality to a proof that their interpretations are related by
\(\AgdaOperator{\AgdaField{\_≈ᴹ\_}}\).  This makes equality a parameter
of the model rather than forcing every model to use Agda's judgmental or
propositional equality.

The small \AgdaModule{Core.Models.Setoids} layer packages these hom
relations as standard-library setoids.  For the first-order
paramorphism models, the construction is:

{\small
\CorePRFOHomSetoid*
}

This packaging is mostly administrative, but it is important for the
shape of the formalization.  It lets later developments reason about
hom-objects extensionally, while the syntax remains available for
structural recursion, translations between presentations, and generated
proofs of equality preservation.  The same pattern is repeated for the
fold-primitive variants and for the higher-order calculi; only the list
of primitive operations and laws changes.
